# Развёртывание сервера (игра + админка призов)

Игра «Растопи лёд» и админка работают на одном Node-сервере. Клиентам даёте ссылку на игру и отдельно — на админку (логи выигрышей, правка призов).

## Что нужно для запуска

- **Node.js** 18+
- Переменная окружения **`ADMIN_PASSWORD`** — пароль входа в админку. Если не задана, админка открывается без пароля (удобно для локальной разработки).

## Запуск локально

```bash
npm install
set ADMIN_PASSWORD=ваш_пароль
node maximumus-presets-server.js
```

Или без пароля (админка без входа):

```bash
node maximumus-presets-server.js
```

- Игра: http://localhost:3000/index.html  
- Админка: http://localhost:3000/ice-admin.html

## Где развернуть для клиентов

### 1. Railway (удобно, есть бесплатный тир)

1. Зарегистрироваться на [railway.app](https://railway.app).
2. New Project → Deploy from GitHub (привязать репозиторий с проектом).
3. В настройках сервиса задать **Root Directory** — папка с `maximumus-presets-server.js` и файлами (если проект в подпапке).
4. **Variables** → добавить `ADMIN_PASSWORD` = пароль для клиента.
5. **Settings** → Generate Domain. Получите URL вида `https://ваш-проект.up.railway.app`.

Клиенту даёте:
- **Игра:** `https://ваш-проект.up.railway.app/index.html`
- **Админка:** `https://ваш-проект.up.railway.app/ice-admin.html` и пароль из `ADMIN_PASSWORD`.

Данные (призы, лог) хранятся на диске контейнера. При передеплое они сбрасываются, если не подключить Volume (в платных планах).

---

### 2. Render

1. [render.com](https://render.com) → New → Web Service.
2. Подключить репозиторий, указать:
   - **Build:** `npm install`
   - **Start:** `node maximumus-presets-server.js`
3. **Environment** → добавить `ADMIN_PASSWORD`.
4. Сервис получит URL вида `https://ваше-имя.onrender.com`.

Минус: на бесплатном плане сервис «засыпает», первый запрос после простоя может быть медленным. Для стабильной работы лучше тариф с постоянным процессом.

---

### 3. VPS (свой сервер)

Подходит, если нужен полный контроль и постоянные данные без ограничений хостинга.

**Пример (Ubuntu):**

```bash
# На сервере
sudo apt update && sudo apt install -y nodejs npm
# или установить Node 20 через nvm
git clone <ваш-репо> && cd led
npm install
export PORT=3000
export ADMIN_PASSWORD=надёжный_пароль
node maximumus-presets-server.js
```

Чтобы процесс не падал и поднимался после перезагрузки — использовать **PM2**:

```bash
npm install -g pm2
pm2 start maximumus-presets-server.js --name ice
pm2 save && pm2 startup
```

Перед сервером лучше поставить **Nginx** как обратный прокси и выдать HTTPS (Let's Encrypt). Тогда клиент заходит по `https://ваш-домен.ru`.

Данные лежат в папке **`data/`**:
- `data/ice-prizes.json` — призы по email
- `data/ice-wins.json` — лог выигрышей  

Их имеет смысл регулярно бэкапить (cron + копия в облако или другой сервер).

## Доступ для клиентов

| Что | Ссылка | Комментарий |
|-----|--------|-------------|
| Игра | `https://ваш-домен/index.html` | Можно раздать участникам |
| Админка | `https://ваш-домен/ice-admin.html` | Только для вас/клиента; вход по паролю |

В админке клиент может:
- смотреть **лог выигрышей** (время, email, приз);
- **редактировать призы**: добавить/изменить/удалить строки (email → название, редкость, номинал, описание);
- ключ **`default`** — приз, который выдаётся, если email не найден в списке.

Пароль задаётся один раз через `ADMIN_PASSWORD` на сервере; его можно передать клиенту отдельно (мессенджер, почта).
